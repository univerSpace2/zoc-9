# ZOC9 족구 규칙 명세 (v1)

- 문서 목적: 경기 기록 엔진과 UI가 동일 규칙을 참조하도록 단일 소스 제공.
- 적용 범위: 매치/세트 생성, 라이브 득점 입력, 종료 판정.

## 1. 경기 방식
1. 한 경기는 아래 방식 중 하나를 사용한다.
   - 단판: 1세트 승부.
   - 3판 2선승제: 최대 3세트, 먼저 2세트 승리 팀 우승.
   - 5판 3선승제: 최대 5세트, 먼저 3세트 승리 팀 우승.
2. 필요한 세트 승수를 먼저 충족하면 즉시 경기 종료한다.
3. 남은 세트는 진행하지 않고 `ignored` 상태로 처리한다.

## 2. 세트 점수 규칙
1. 각 세트 목표 점수는 설정 가능하며 기본값은 15점이다.
2. 듀스 적용 여부를 선택할 수 있으며 기본은 적용(on)이다.
3. 듀스 on:
   - 목표점수 15점 기준 양 팀 14:14가 되면 듀스 상태다.
   - 이후 2점 차가 나는 즉시 세트 종료.
4. 듀스 off:
   - 목표점수에 먼저 도달한 팀이 세트 승리.

## 3. 세트 인원 구성
1. 팀당 인원 수는 2인, 3인, 4인 중 하나를 선택한다.
2. 인원 수에 따라 포지션 수와 로테이션 순서가 달라진다.

## 4. 포지션 및 서브 로테이션 순서
### 4인 경기
1. 우수비 (1번)
2. 좌수비 (2번)
3. 공격수 (3번)
4. 세터 (4번)

### 3인 경기
1. 수비 (1번)
2. 공격수 (2번)
3. 세터 (3번)

### 2인 경기
1. 수비 (1번)
2. 공격수 (2번)

## 5. 세트 시작 규칙
1. 경기 시작 전에 1세트 최초 서브 팀을 지정한다.
2. 1세트는 지정된 팀이 서브한다.
3. 다음 세트의 최초 서브 팀은 직전 세트에서 최초 서브를 했던 팀의 반대 팀이다.
4. 모든 세트는 시작 시 초기 서브 포지션을 1번으로 리셋한다.
5. 세트 간 로테이션 진행 상태는 이어지지 않는다.

## 6. 서브권 변경 및 로테이션
1. 서브 중인 팀이 실점하면 서브권은 상대 팀으로 넘어간다.
2. 로테이션은 서브권을 새로 획득한 시점에만 수행한다.
3. 서브권을 유지하며 득점한 경우 로테이션은 수행하지 않는다.
4. 로테이션 상태는 팀별로 독립적이며 상대 팀의 로테이션에 영향을 주지 않는다.

## 7. 세트 종료 판정
1. 듀스 off: 목표점수 도달 즉시 종료.
2. 듀스 on:
   - 양 팀 점수가 `(목표점수 - 1)` 이상인 동점 구간에 진입한 뒤,
   - 2점 차가 발생하면 종료.

## 8. 매치 종료 판정
1. 경기 방식별 필요 승수에 먼저 도달한 팀을 승리팀으로 확정한다.
2. 승리 확정 후 남은 세트는 `ignored`로 상태 변경한다.

## 9. 이벤트 기록 규칙
1. 득점 입력은 `set_events`에 이벤트 단위로 저장한다.
2. 이벤트는 `client_event_id`로 멱등 처리한다.
3. 서버 검증 결과가 최종 정본이며, 클라이언트는 낙관적 UI를 사용하더라도 서버 상태로 재동기화한다.
